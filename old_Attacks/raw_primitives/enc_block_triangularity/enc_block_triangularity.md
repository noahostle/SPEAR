# Exact Byte-Triangular Collapse of `ENC_Block` in SEPAR

## Statement

For every 32-bit segment key `K = (k0, k1)` and every rotor index `n in {1, ..., 8}`,
the implemented 16-bit core `ENC_Block` is exactly byte-triangular:

```text
ENC_Block_K,n(u || l) = U_K,n(u) || L_K,n(u, l)
```

where `u` is the input upper byte and `l` is the input lower byte.

Equivalently:

```text
output_upper_byte depends only on input_upper_byte.
```

So varying the lower input byte while holding the upper input byte fixed can never change
the upper output byte.

This is an exact structural collapse of the intended 16-bit diffusion layer.

## Where it comes from in the implementation

The property follows from the exact form of the linear layer and the round structure. 

`ENC_Block` applies:

1. xor with a 16-bit subkey
2. four parallel 4-bit S-boxes
3. `Sep_ROTL16`

four times, then a final key xor / S-box / key xor output layer.

The crucial point is that the implemented `Sep_ROTL16` is much weaker than the design
narrative suggests.

## Exact linear-layer collapse

Write a 16-bit word as four nibbles

```text
X = (a, b, c, d)
```

from most significant to least significant.

The code first performs:

```text
a <- a xor c
b <- b xor d
c <- c xor b
d <- d xor a
```

and then xors the result with `ROTL16(X, 12)` and `ROTL16(X, 8)`.

A direct nibble calculation simplifies the full implemented map to:

```text
L(a, b, c, d) = (b xor c, b, a, a xor d).
```

So the second nibble of the output is just the previous second nibble after the S-box
layer, and the upper byte only depends on the old middle two nibbles.

## Dependency recurrence through one round

Let the inputs to one S-box layer be `(u_r, v_r, w_r, z_r)` and let the corresponding
S-box outputs be

```text
A_r = S1(u_r)
B_r = S2(v_r)
C_r = S3(w_r)
D_r = S4(z_r).
```

After the linear layer and next key xor, the next-round S-box inputs satisfy:

```text
u_{r+1} = B_r xor C_r xor k^u_{r+1}
v_{r+1} = B_r xor k^v_{r+1}
w_{r+1} = A_r xor k^w_{r+1}
z_{r+1} = A_r xor D_r xor k^z_{r+1}.
```

Conseqeunces are:

- `v_{r+1}` depends only on `v_r`
- `w_{r+1}` depends only on `u_r`
- `u_{r+1}` depends only on `(v_r, w_r)`

There is no dependence on `z_r` in the upper part of the next round.

## Exact dependency propagation

Start from the input nibble tuple

```text
(u_1, v_1, w_1, z_1) = (a, b, c, d).
```

Track only which input nibbles can influence each upper-lane variable:

```text
u_2 <- {b, c}     v_2 <- {b}     w_2 <- {a}
u_3 <- {a, b}     v_3 <- {b}     w_3 <- {b, c}
u_4 <- {b, c}     v_4 <- {b}     w_4 <- {a, b}
u_5 <- {a, b}     v_5 <- {b}
```

The final output layer is only an xor, S-box layer, and xor. It does not reintroduce any
dependence from the lower input byte into the upper output byte.

Therefore the final upper byte depends only on the initial upper byte `(a, b)`.

## Consequences

### `ENC_Block` preserves an exact 8-bit invariant partition

The 16-bit block space is partitioned into 256 classes indexed by input upper byte, and
each class is mapped into a single output upper-byte class.

So `ENC_Block` is not behaving like a diffused 16 bit permutation. its a triangular map with only 8 effective bits of diffusion in its upper half.

### The lower input byte cannot influence the upper output byte inside a block

### interstage diffusion is reduced to carry branching

In the full cipher, each stage looks like

```text
x -> x + state_i -> ENC_Block_i(x + state_i).
```

Since `enc_block_i` is triangular, the only way the lower byte affects the upper-byte
process is through the carry bit generated by the addition `x + state_i`.

contains a Python reimplementation of `ENC_Block` from the original spec.

It performs:

- an exhaustive check that the closed form linear layer matches `Sep_ROTL16`
- exhaustive byte triangularity checks over all `2^16` inputs for sampled keys
